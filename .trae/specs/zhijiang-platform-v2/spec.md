# 智匠智装平台 v2 - 产品需求文档

## Overview
- **Summary**: 智匠智装平台 v2 是对现有平台的功能扩展，增加自定义服务项目、完善聊天功能和支付功能，提升用户体验和平台完整性。
- **Purpose**: 解决平台功能不足的问题，提供更灵活的服务选择、更便捷的沟通方式和更安全的支付流程。
- **Target Users**: 
  - 客户：需要技术帮助的个人或企业
  - 工程师：提供技术服务的专业人士
  - 管理员：平台运营和管理工作人员

## Goals
- 增加自定义服务项目，限制在 AI 和 coding 范围内
- 完善聊天功能，支持已接触客户的实时沟通
- 实现类似闲鱼的支付流程，确保交易安全
- 提升平台的用户体验和功能性

## Non-Goals (Out of Scope)
- 实时视频通话功能
- 复杂的评价系统
- 多语言支持
- 移动端 App 开发

## Background & Context
- 平台已完成基础架构搭建，包括用户注册、登录、需求提交、工程师接单等功能
- 目前存在的问题：服务项目固定、聊天功能简单、缺乏支付系统
- 需要扩展平台功能，提升用户体验和平台竞争力

## Functional Requirements
- **FR-1**: 自定义服务项目
  - 在服务专区增加一种自定义项目类型
  - 限制自定义项目在 AI 和 coding 范围内
  - 修改提需求部分，支持自定义项目的需求提交
- **FR-2**: 完善聊天功能
  - 支持已接触客户的实时聊天
  - 提供聊天历史记录
  - 支持消息通知
- **FR-3**: 支付功能
  - 实现类似闲鱼的支付流程
  - 用户支付到平台账户
  - 平台审核后支付给工程师
  - 支持退款功能

## Non-Functional Requirements
- **NFR-1**: 响应式设计
  - 聊天功能在桌面端和移动端都能良好显示
  - 支付流程适配不同屏幕尺寸
- **NFR-2**: 安全性
  - 支付信息加密传输
  - 防止支付欺诈
  - 保护用户敏感信息
- **NFR-3**: 性能
  - 聊天消息实时性
  - 支付流程响应及时
  - 页面加载速度快
- **NFR-4**: 可扩展性
  - 支付系统支持多种支付方式
  - 聊天功能支持未来扩展（如表情、文件传输等）

## Constraints
- **Technical**: Next.js 16.2.2, TypeScript, TailwindCSS, Supabase
- **Business**: 优先保证功能可运行，不追求复杂实现
- **Dependencies**: Supabase Auth, PostgreSQL, Storage, 第三方支付 API

## Assumptions
- 用户已经完成注册和登录
- 工程师已经通过平台审核
- 平台已经配置好数据库和环境变量
- 已经申请并获得支付 API 密钥

## Acceptance Criteria

### AC-1: 自定义服务项目功能
- **Given**: 用户访问服务专区页面
- **When**: 选择自定义项目类型
- **Then**: 可以填写自定义需求，限制在 AI 和 coding 范围内
- **Verification**: `human-judgment`
- **Notes**: 自定义项目应包含详细的需求描述字段

### AC-2: 聊天功能完善
- **Given**: 客户和工程师已经建立联系
- **When**: 访问聊天页面
- **Then**: 可以实时发送和接收消息，查看聊天历史
- **Verification**: `human-judgment`
- **Notes**: 聊天功能应支持基本的文本消息

### AC-3: 支付功能实现
- **Given**: 工程师完成任务
- **When**: 客户确认完成并支付
- **Then**: 资金进入平台账户，平台审核后支付给工程师
- **Verification**: `human-judgment`
- **Notes**: 支付流程应清晰易懂，确保资金安全
